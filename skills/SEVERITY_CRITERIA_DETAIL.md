# 보안 취약점 위험도 등급 상세 기준 (5~1)

이 문서는 취약점별 상세 등급 기준을 정의한다.

보안 취약점 위험도 등급 정의

- 5등급 매우 위험(Critical): 공격 성공 시 즉시 금전 손실, 계정 탈취, 시스템 장악 등 심각한 피해 가능성이 매우 높음 (예: 원격 코드 실행, 인증 우회, 거래 위변조, 세션 탈취 등)
- 4등급 고위험(High): 직접적인 피해로 이어질 가능성이 높으며, 민감정보 노출/권한 상승/서비스 악용이 가능한 취약점 (예: XSS, CSRF, 인증 우회, 디렉토리 인덱싱 등)
- 3등급 중간 위험(Medium): 설정 오류·구성 문제로 보안 수준을 약화시키며, 다른 취약점과 결합해 악용될 가능성이 있음 (예: 취약한 HTTPS 구성, 인증서 오류, 백그라운드 정보 노출 등)
- 2등급 저위험(Low): 직접적 영향은 낮으나 특정 조건에서 악용될 수 있음 (예: 재협상 취약한 HTTPS 설정 등)
- 1등급 매우 낮음(Informational): 악용까지의 거리가 멀거나 운영상 거의 영향이 없는 취약점 (예: 불필요한 메서드 허용 등)
